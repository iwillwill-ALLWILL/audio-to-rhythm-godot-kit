# Audio to Rhythm Godot Kit

把一首音乐变成可以放进你自己 Godot 游戏里的音游关卡。

这个仓库不是给普通用户手动研究代码的。正确用法是：**把仓库、Godot 项目、音乐文件交给 AI，让 AI 自动接入、生成、验证。**

---

## 你和 AI 只做三件事

### 1. 部署引擎

第一次用，先让 AI 把音游引擎接进你的 Godot 项目。

直接把这段发给 AI：

```text
这是我的 Godot 游戏项目：<填你的项目路径>
这是音游引擎仓库：https://github.com/iwillwill-ALLWILL/audio-to-rhythm-godot-kit

请把这个音游引擎部署到我的游戏里。
要求：
- 不要破坏我原来的游戏内容；
- 先做一个可以进入音游的测试入口；
- 跑 Godot 验证，报错就继续修；
- 最后告诉我从哪里进入音游。
```

完成后，你的游戏里就有一个可运行的音游入口。

---

### 2. 传音乐，选择难度

引擎接好后，你只需要继续给 AI 音乐文件和难度。

```text
这是我的 Godot 游戏项目：<填你的项目路径>
这是音乐文件：<填音乐文件路径>
关卡名：<填名字，或者让 AI 自己起>
难度：<easy / normal / hard / expert / 自定义要求>

请把这首歌生成成音游关卡，加入到已有音游入口里。
完成后跑 Godot 验证，报错就继续修。
最后告诉我新关卡从哪里进入。
```

难度可以很简单：

```text
easy     轻松
normal   普通
hard     更密
expert   当前最高难度，包含更多双键和长按
```

---

### 3. 个性化引擎

如果默认玩法不符合你的游戏，就让 AI 改成你的风格。

```text
我的游戏风格是：<做饭 / 战斗 / 跑酷 / 剧情 / 经营 / 解谜 / 角色互动 / 其他>
我希望音游入口放在：<主菜单 / 关卡选择 / NPC 对话 / 地图触发器 / 测试按钮>
我希望按键和反馈表现成：<例如切菜、攻击、跳跃、施法、对话、演出等>
我希望结算和奖励是：<例如分数、星级、道具、剧情推进、解锁新关卡>

请按这些要求个性化音游引擎，并跑 Godot 验证。
```

也就是说，你不需要研究怎么改代码。你只需要告诉 AI：**放在哪里、长什么样、怎么玩、怎么奖励。**

## 想先试一个现成 demo？

仓库里已经放了一个可直接打开的 Godot demo：

```text
examples/ready_expert_godot
```

这个 demo 使用真实人声音乐，难度是 `expert`，也就是当前内置最高难度。

---

## 你不需要做什么

你不需要：

- 自己写代码；
- 自己看谱面文件；
- 自己研究 Godot 导入；
- 自己调音游参数；
- 自己处理报错。

这些都交给 AI。
